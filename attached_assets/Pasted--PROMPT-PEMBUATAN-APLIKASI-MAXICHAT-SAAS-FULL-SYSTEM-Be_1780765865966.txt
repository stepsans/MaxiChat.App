# SYSTEM OVERVIEW

MaxiChat adalah aplikasi Omnichannel SaaS.

Setiap pelanggan disebut Membership.

Setiap Membership memiliki Parent User sebagai Super Admin.

Parent dapat membuat Child User.

Role yang tersedia:

1. Parent (Super Admin)
2. Supervisor
3. Agent

---

# USER HIERARCHY

Parent
├── Supervisor
└── Agent

Hak akses Parent:

* Mengelola subscription
* Mengelola billing
* Menambah Supervisor
* Menambah Agent
* Menghapus Supervisor
* Menghapus Agent
* Melihat seluruh laporan
* Mengatur channel
* Mengatur AI

Hak akses Supervisor:

* Monitoring Agent
* Melihat seluruh chat
* Assign chat
* Melihat laporan

Hak akses Agent:

* Menangani chat
* Menggunakan AI Assistant
* Mengubah status chat

Implementasikan RBAC yang fleksibel dan scalable.

---

# LOGIN SYSTEM

Metode login:

Email OTP

Flow:

1. User memasukkan email
2. Sistem mengirim OTP ke email
3. OTP berlaku 5 menit
4. Maksimal 5 kali percobaan
5. OTP berhasil → Login
6. Generate JWT Access Token
7. Generate Refresh Token

Tambahkan:

* Rate Limiting
* Anti Spam OTP
* Audit Log

---

# INVITATION SYSTEM

Parent dapat mengundang Supervisor atau Agent.

Flow:

1. Parent klik Invite User
2. Input email
3. Pilih role
4. Sistem kirim email invitation
5. Email berisi tombol Accept Invitation
6. Link memiliki token unik dan expiry

Status invitation:

* Pending
* Accepted
* Rejected
* Expired

Saat invitation diterima:

* User dibuat otomatis
* Role diberikan otomatis
* Tenant diberikan otomatis

---

# MULTI TENANT ARCHITECTURE

Setiap Membership adalah Tenant.

Setiap Tenant memiliki:

* User
* Chat
* Channel
* Billing
* AI Usage
* Subscription

Data Tenant wajib terisolasi.

Tidak boleh ada kemungkinan tenant melihat data tenant lain.

Gunakan tenant_id pada seluruh tabel bisnis.

Tambahkan middleware tenant validation.

# SUBSCRIPTION SYSTEM

Subscription bersifat dinamis.

Harga tidak boleh hardcoded.

Harga wajib diambil dari database.

Buat Pricing Engine.

---

# PRICING COMPONENT

1. Database Chat

Default:

500 MB = Rp 50.000 / bulan

---

2. Child User

1 Child User = Rp 50.000 / bulan

Yang dihitung:

* Supervisor
* Agent

Parent tidak dihitung.

---

3. Channel

2 Channel = Rp 50.000 / bulan

Contoh:

2 Channel = Rp 50.000

4 Channel = Rp 100.000

6 Channel = Rp 150.000

---

4. AI Token

100 Token = Rp 50.000 / bulan

Harus configurable.

---

# BILLING ENGINE

Buat sistem billing fleksibel.

Formula:

Monthly Bill =
Database Charge
+
User Charge
+
Channel Charge
+
AI Token Charge

Semua parameter berasal dari database.

Admin dapat mengubah harga kapan saja tanpa deploy ulang aplikasi.
