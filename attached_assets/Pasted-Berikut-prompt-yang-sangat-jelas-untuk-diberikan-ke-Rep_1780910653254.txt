# PROMPT UNTUK REPLIT AI

Saya ingin melakukan audit sistem penggunaan token AI di MaxiChat.

⚠️ IMPORTANT:
JANGAN mengubah kode apa pun.
JANGAN membuat migration database.
JANGAN mengubah schema.
JANGAN mengubah logic aplikasi.

Mode hanya ANALISIS dan LAPORAN.

## Tujuan

Saya ingin memahami secara detail bagaimana token AI dihitung dan dicatat di dalam aplikasi MaxiChat.

Tolong periksa seluruh source code dan buat laporan lengkap mengenai:

### 1. Sumber Data Token

Untuk menu penggunaan token yang saat ini menampilkan:

* Total Token
* Prompt Token
* Completion / Response Token
* Jumlah Panggilan AI (AI Calls)

Jelaskan:

* Data tersebut berasal dari mana
* File mana yang menghitungnya
* API mana yang mengirim data tersebut
* Database tabel mana yang menyimpannya
* Apakah data tersebut real dari vendor AI atau estimasi internal

---

### 2. Apakah Token Development Ikut Terhitung?

Periksa apakah penggunaan AI berikut ikut masuk statistik token:

* AI yang digunakan developer saat membangun aplikasi
* AI yang digunakan Replit Agent
* AI yang digunakan Claude Code
* AI internal untuk debugging
* AI internal untuk testing

atau

Apakah hanya penggunaan AI dari end-user MaxiChat yang dihitung.

Buat kesimpulan yang jelas:

✅ Hanya User Tenant

atau

⚠️ Campur antara User dan Development

### 3. Tenant Isolation

Periksa apakah token usage sudah terikat dengan tenant masing-masing.

Contoh:

Tenant A:

* 10.000 token

Tenant B:

* 25.000 token

Apakah sistem saat ini:

A. Menyimpan token berdasarkan tenant_id

atau

B. Semua token tercampur global

Jelaskan:

* Field database yang digunakan
* Query yang digunakan
* Risiko jika saat ini belum terisolasi

### 4. Akurasi Perhitungan

Jelaskan secara detail.
